# Симулятор хаотических систем

👉 **[Инструкция по установке](INSTALL.md)** — если Julia ещё не стоит

Интерактивная программа для численного интегрирования и анализа нелинейных динамических систем. Запускается через GLMakie — открывает окно с графиками и панелью управления, где можно менять параметры в реальном времени.

## Что умеет

- Интегрирует пять классических систем: аттрактор Лоренца, хищник-жертва, осциллятор Ван дер Поля, аттрактор Росслера, двойной маятник
- Четыре численных метода на выбор: Эйлер, Хойн (RK2), Рунге-Кутта 4-го порядка, адаптивный шаг
- Показатель Ляпунова — числовой ответ на вопрос «это хаос или нет»
- Три режима визуализации: 3D траектория, фазовый портрет, временны́е ряды
- Экспорт текущего вида в PNG

## Быстрый старт

```julia
julia Main.jl
```

Требования: Julia 1.9+, пакет `GLMakie`. Если пакет не установлен:

```julia
using Pkg
Pkg.add("GLMakie")
```

## Структура

```
Main.jl       — точка входа
Gui.jl        — окно, слайдеры, реактивные графики
Models.jl     — уравнения систем
Solvers.jl    — численные методы
Analysis.jl   — показатель Ляпунова, статистика, сохранение
```

Зависимости между модулями однонаправленные: `Gui` → `Models`, `Solvers`, `Analysis`. Модули нижних уровней не знают о GUI ничего.

## Модели (`Models.jl`)

Каждая система — Julia-структура с параметрами. Все системы реализуют один интерфейс:

| Функция | Что возвращает |
|---|---|
| `get_derivative(sys, u)` | вектор производных `du/dt` |
| `system_name(sys)` | строка с названием |
| `system_dimension(sys)` | число уравнений |
| `default_u0(sys)` | начальные условия по умолчанию |
| `default_params(Type)` | параметры по умолчанию |

### Аттрактор Лоренца

```
dx/dt = σ(y − x)
dy/dt = x(ρ − z) − y
dz/dt = xy − βz
```

Модель тепловой конвекции 1963 года. При σ=10, ρ=28, β=8/3 — детерминированный хаос. Классические параметры: `LorenzSystem(10.0, 28.0, 8/3)`.

### Лотка–Вольтерра

```
dx/dt =  αx − βxy
dy/dt = −γy + δxy
```

Модель «хищник–жертва». Решения периодические — замкнутые орбиты в фазовом пространстве, хаоса нет. Существует интеграл движения `V = δx − γ ln x + βy − α ln y = const`, по дрейфу которого можно судить о точности метода.

### Осциллятор Ван дер Поля

```
dx/dt = y
dy/dt = μ(1 − x²)y − x
```

Нелинейный осциллятор с автоматической стабилизацией амплитуды. При |x| < 1 система «накачивается», при |x| > 1 — тормозится. Результат — устойчивый предельный цикл. При μ → 0 колебания почти синусоидальные, при μ >> 1 — релаксационные (резкие скачки).

### Аттрактор Росслера

```
dx/dt = −y − z
dy/dt =  x + ay
dz/dt =  b + z(x − c)
```

Придуман в 1976 году как упрощение Лоренца. Механизм хаоса наглядный: спираль в плоскости XY растёт до порогового значения x = c, после чего компонента z резко выбрасывается вверх и траектория «складывается» обратно.

### Двойной маятник

Две точечные массы на шарнирах. Состояние: `[θ₁, ω₁, θ₂, ω₂]` — углы и угловые скорости. Уравнения выведены из лагранжевой механики. При начальных углах больше ~20° быстро переходит в хаос — наиболее чувствительная к начальным условиям система из всех пяти.

## Численные методы (`Solvers.jl`)

Все солверы возвращают `(t_steps, u_results, stats)`, где `stats :: SolverStats` содержит число вызовов производной, число отклонённых шагов, диапазон использованных шагов.

### Метод Эйлера (порядок 1)

```
u[i+1] = u[i] + dt · f(u[i])
```

Один вызов производной на шаг. Глобальная ошибка O(dt). Годится только для демонстрации — на хаотических системах быстро расходится.

### Метод Хойна — RK2 (порядок 2)

```
k1 = f(u)
k2 = f(u + dt · k1)
u[i+1] = u[i] + dt/2 · (k1 + k2)
```

Два вызова производной. Сначала делает шаг Эйлером (предиктор), потом усредняет производные в начале и конце шага — это правило трапеции. Глобальная ошибка O(dt²).

### Рунге–Кутта 4-го порядка (порядок 4)

```
k1 = f(u)
k2 = f(u + dt/2 · k1)
k3 = f(u + dt/2 · k2)
k4 = f(u + dt · k3)
u[i+1] = u[i] + dt/6 · (k1 + 2k2 + 2k3 + k4)
```

Четыре вызова производной. Веса 1,2,2,1 соответствуют правилу Симпсона. Глобальная ошибка O(dt⁴). Стандартный выбор для большинства задач.

### Адаптивный метод (пара Хойн–Эйлер)

На каждом шаге вычисляет два решения — первого и второго порядка — и сравнивает их. Разность служит оценкой локальной ошибки:

```
если ошибка ≤ tol  → шаг принят, следующий шаг увеличить
если ошибка > tol  → шаг отклонён, уменьшить и пересчитать
```

Формула изменения шага: `dt_new = dt · clamp(0.9 · (tol/error)^0.5, 0.2, 5.0)`. Коэффициент 0.9 — запас надёжности, пределы 0.2/5.0 защищают от слишком резких изменений. Полезен, когда траектория чередует плавные и резкие участки.

**Сравнение методов** — утилита `Solvers.compare_methods(sys, u0, tspan, dt)` запускает все четыре метода и печатает таблицу: число вызовов производной, число шагов, ошибка относительно эталона (RK4 с шагом dt/10).

## Анализ (`Analysis.jl`)

### Показатель Ляпунова

Мера экспоненциальной чувствительности к начальным условиям. Алгоритм с перенормировкой:

1. Запустить две траектории: основную и с малым возмущением δ = ε в первой компоненте
2. Через каждые `n_renorm` шагов измерить расстояние d между ними
3. Накопить сумму: `λ_sum += log(d / ε)`
4. Сбросить возмущение обратно к ε, сохранив направление: `u_pert = u + diff · (ε / d)`
5. Итог: `λ = λ_sum / (n_renorm_steps · dt · n_renorm)`

Интерпретация:
- λ > 0 — хаос, горизонт предсказуемости ≈ 1/λ единиц времени
- λ ≈ 0 — граница хаоса
- λ < 0 — устойчивые орбиты

```julia
λ = Analysis.lyapunov_exponent(sys, u0, dt; n_steps=10_000, epsilon=1e-8)
```

### Статистика траектории

`trajectory_stats(u_results, dt)` возвращает структуру `TrajectoryStats`:

- `mean_u`, `std_u` — среднее и стандартное отклонение по каждой координате
- `min_u`, `max_u` — диапазоны
- `mean_speed`, `max_speed` — скорость движения в фазовом пространстве
- `path_length` — суммарная длина пути

### Скорость расхождения

`divergence_rate(sys, u0, dt)` возвращает `(t_steps, distances)` — расстояние между двумя близкими траекториями без перенормировки. На хаотической системе кривая растёт экспоненциально до насыщения (размер аттрактора), на периодической — остаётся ограниченной.

### Проверка сохраняющихся величин

Для систем с интегралом движения — числовой контроль точности солвера.

**Лотка–Вольтерра:** интеграл `V = δx − γ ln x + βy − α ln y`  
**Двойной маятник:** полная механическая энергия `E = T + U`

```julia
values, drift = Analysis.conservation_check(sys, u_results)
# drift — относительный дрейф (V_max - V_min) / |V_mean|
# drift < 0.001 → солвер точный
```

### Сводный отчёт

```julia
Analysis.print_analysis_report(sys, u_results; dt=0.01)
```

Печатает в консоль: размеры траектории, диапазоны координат, показатель Ляпунова с интерпретацией, дрейф интеграла (если применимо).

## Интерфейс (`Gui.jl`)

Построен на реактивных `Observable` из GLMakie. Любое изменение в панели управления автоматически пересчитывает траекторию и обновляет все три графика.

**Вкладки:**
- *3D траектория* — пространственный вид, полезен для трёхмерных систем
- *Фазовый портрет* — проекция на плоскость (X, Y)
- *Временны́е ряды* — X(t) и Y(t) на одном графике

**Цветовая карта** — по умолчанию кривая окрашена по скорости движения (медленные участки тёмные, быстрые — яркие). Можно переключить на однородный цвет.

**Экспорт:** кнопка «Сохранить PNG» записывает текущий вид в файл рядом со скриптом. Имя файла формируется автоматически из названия системы.

## Добавление новой системы

1. В `Models.jl` определить структуру и реализовать пять функций интерфейса
2. В `Gui.jl` добавить строку в `SYSTEMS`, ветку в `make_system` и `u_results_obs`, начальные значения слайдеров в `on(system_menu.selection)`

Больше ничего менять не нужно — солверы и анализ работают с любым типом через multiple dispatch.

## Примеры из REPL

```julia
include("Models.jl");   using .Models
include("Solvers.jl");  using .Solvers
include("Analysis.jl"); using .Analysis

# Интегрировать аттрактор Лоренца
sys = LorenzSystem(10.0, 28.0, 8/3)
t, u, stats = solve_rk4(sys, default_u0(sys), (0.0, 50.0), 0.01)
println(stats)

# Показатель Ляпунова
λ = lyapunov_exponent(sys, default_u0(sys), 0.01)
println("λ = $λ")   # ожидается ~0.9 для классических параметров

# Сравнение методов
compare_methods(sys, default_u0(sys), (0.0, 10.0), 0.01)

# Полный отчёт
print_analysis_report(sys, u; dt=0.01)
```
